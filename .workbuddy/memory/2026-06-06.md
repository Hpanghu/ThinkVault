# 2026-06-06 工作日志

## BM25 索引持久化实现

### 新文件
- `thinkvault/core/bm25_index_store.py` — BM25 索引磁盘持久化存储
  - SQLite 元数据表 + JSON 文件存储索引数据
  - `save_index()`: 原子写入（tmp → rename），同步更新 SQLite 元数据
  - `load_index()`: 版本检查 + chunk 数量校验，不匹配自动失效
  - `delete_index()`, `delete_all()`, `list_indexes()`, `get_index_meta()`
  - 修复: `save_index()` 中增加 `_ensure_index_dir()` 调用，确保目录存在

### 修改文件
- `retriever.py` — BM25 持久化集成
  - 新增 `_BM25_PERSIST` 环境变量（THINKVAULT_BM25_PERSIST，默认 1）
  - `_get_bm25()` 重写：内存缓存 → 磁盘缓存 → ChromaDB 全量构建（三级降级）
  - 新增 `_load_bm25_from_disk()`: 从磁盘加载 + `_reconstruct_bm25()` 重建 BM25Okapi 对象
  - 新增 `_save_bm25_to_disk()`: 提取 BM25Okapi 内部参数并持久化
  - `invalidate_cache()`: 清理内存缓存 + 同步删除磁盘持久化索引
- `container.py` — 注册 `bm25_index_store` 服务
