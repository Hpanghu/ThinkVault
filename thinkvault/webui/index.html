操作指引

模型文件路径

~/.thinkvault/models/
  • 将 .gguf 格式的模型文件放入此目录
  • 系统启动时自动扫描该目录下所有 .gguf 文件
  • 推荐使用 Q4_K_M 量化格式，兼顾速度与质量
~/.thinkvault/
├── models/              ← 模型文件目录
│   ├── qwen2.5-0.5b-instruct-q4_k_m.gguf
│   └── your-model.gguf
├── data/                ← 知识库数据
│   └── thinkvault.db
└── logs/                ← 运行日志
    └── llama-server.log

知识库数据存储

~/.thinkvault/data/thinkvault.db
  • 使用 SQLite 数据库存储，单文件，便于备份
  • 支持格式：PDF、TXT、Markdown、DOCX
  • 文档自动分块索引，混合检索（向量 + 关键词）
  • 访问权限：仅本机用户可读写，无网络暴露

快速上手流程

1
启动推理服务

点击左侧栏「一键启动本地服务」按钮，系统自动扫描模型并启动推理引擎。也可手动选择模型后点击「启动本地服务」。

2
上传文献

点击左侧「文献」区域的上传按钮，或聊天输入框旁的附件按钮，支持 PDF/TXT/MD/DOCX 格式。

3
向馆长提问

在输入框中输入问题，系统自动检索相关文献片段，结合 AI 生成精准回答。回答会标注引用来源。

4
管理知识库

在左侧「对话」区域切换知识库，不同主题的文献可归入不同知识库独立管理。

远程接入流程

A
切换到「远程接入」模式

在左侧栏模型设置区域，点击「远程接入」标签页。

B
填写后端地址

输入局域网内推理服务的地址，如 http://192.168.1.100:8080/v1

C
配置 API Token

如远程服务启用了认证，填写对应的 API Token。无需认证则留空。

D
点击「连接远程」

系统自动检测连接，成功后状态栏变为绿色在线状态。

数据安全提示

完全离线运行

所有数据存储和处理均在本地完成，不会上传至任何云端服务器。模型推理和文档索引全部在本机执行。

数据不出设备

文档内容、对话记录和模型权重始终保存在本机，无任何外部网络请求（远程接入模式除外）。

远程接入注意事项

使用远程接入时，数据会通过局域网传输至远程推理服务。请确保网络环境可信，并配置 API Token 进行认证保护。

定期备份数据

知识库数据存储在 ~/.thinkvault/data/ 目录，建议定期备份 thinkvault.db 文件以防数据丢失。

API Token 认证

生产环境建议设置 THINKVAULT_API_TOKEN 环境变量启用认证，防止未授权访问。开发模式可设置 THINKVAULT_DISABLE_AUTH=1 跳过认证。
